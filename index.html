SPT Planning Calendar Builder Guide

Fill in the panel on the left and the schedule builds itself in the preview. Everything below explains what each control does and how the calendar is calculated. Nothing you enter leaves your computer.

Show Info

Title names the project. It appears in the printed Excel header and becomes the file name when you export or save (e.g. “Show Title - Season 1 Planning Calendar.xlsx”).

Shooting Days per Episode is informational context for the shoot — it’s shown in the header’s production summary. The actual shoot length is driven by the total shooting-day count you set on the Production phase.

Phases

Each phase is a block of the schedule — Writer’s Room, Pre Prep, Prod Prep, Production, and Post are provided by default, each with its own color. For every phase you set:

  • Start date — any date snaps to the Monday of that week (weeks always run Monday–Sunday).
  • Duration — in weeks for most phases. Production is special: it’s measured in shooting days, not weeks, and is scheduled day-by-day (see Union holidays below). Its total isn’t typed — it’s the sum of the episodes, which come from Show Info.
  • Name — click the phase name to rename it; the label carries through to every week of that phase and to the export.
  • Color — click the little color square to the left of any phase name (built-in or custom) to pick from the standard palette. The choice applies everywhere: the Waterfall, the Month view, and the export.
  • Phase hiatus — turn on the toggle under any phase (e.g. “Writer’s Rm Hiatus”) to pause just that phase for a set number of weeks from a chosen Monday. It works like the all-phase hiatus — the phase still delivers its full requested weeks — but the pause stays in that phase’s own column and never widens into the others. Click a phase-hiatus band in the Waterfall to rename it or recolor that week.

+ Add phase creates a custom phase with its own name, color, start, and length. Phases can overlap — when two or more run in the same week, the preview and export add columns so they sit side by side. Columns follow a clear rule: the earliest-starting phase takes the leftmost column and keeps it, later overlapping phases sit to its right, and non-overlapping phases share a column to keep the sheet narrow. When only one phase is running in a week, it spans across the empty columns so there are no blank gaps.

Header (Auto vs. Manual). By default the calendar header auto-fills from your inputs (title, production span, Principal Photography dates, episode count) and updates live. Click Header: Auto — Switch to Manual above the calendar to take control: it snapshots the current values into editable lines you can freely edit, and stops auto-updating. Switch back to Auto to discard the manual edits and resume live fill.

Episodes

Fill in Season, Shooting Days per Episode and Number of Episodes in Show Info and the episode list builds itself under Production — named by the usual convention (season 1 gives 101, 102…; season 4 gives 401, 402…). Until those three are filled in, Production can’t be scheduled and its row is greyed out.

Production’s Total Shooting Days is the sum of the episode list, so it’s shown rather than typed. Change any episode’s day count and the total follows — eight 8‑day episodes is 64 shooting days, but bump one to 10 and it becomes 66. Show Info flags any episode you’ve changed by hand, which is what explains a total that isn’t a round multiple. Everything downstream — the Waterfall view, the Excel export, the day-by-day scheduling around weekends, holidays and hiatus — works off that single number exactly as before.

Episodes are added and removed only by changing Number of Episodes. Lowering it removes episodes from the end; if one of those had been edited by hand, the panel says so rather than dropping it silently. In Month view the episodes replace Production’s bar, laid across its shooting days back to back — so an episode can end mid‑week and the next start the same week.

Union holidays

Schedule effect — Production only. Pick a Production Region in the Show tab and, if a recognized union holiday lands on what would have been a shoot day, that day is skipped and the shoot extends day-by-day until the full shooting-day count is met. Holidays never change any other phase’s dates.

Notes display — all phases. Every holiday that lands inside any phase is logged in the Notes column with its date. Use the per-holiday checklist under Union holidays to choose which appear in each view — display only, this doesn’t change the schedule. By default none show in the Waterfall and all show in the Month calendar.

Editing locks the Region. Once you manually edit any comment, holiday, or hiatus band (text, color, or clearing one), the Region selector locks — switching regions would recompute the holidays and discard your edits. Click Reset Notes & Hiatus above the calendar to restore the auto defaults, which unlocks the selector so you can switch regions again.

Simultaneous post

The Simultaneous Post checkbox sits under the Production phase, since that’s what it’s measured against: tick it to mark weeks where post runs concurrently with the shoot, and the offset field beneath sets how many weeks after Production begins the flag starts. A flagged week shows a “Simultaneous Post wk n” marker tucked into the leftmost free column right beside the running phase, so it takes as little space as possible. Those weeks are counted as post weeks — so if two weeks run concurrently (Simultaneous Post wk 1–2), the regular Post phase picks up where they left off and starts at Post wk 3.

All-phase hiatus

Add one or more hiatus periods (a start date and a number of weeks) to pause the whole production at once. A hiatus pushes whatever phase it falls inside outward so that phase still delivers its full requested week or day count — it doesn’t eat into the work. Hiatus weeks appear as a full-width “Hiatus” band. Click any hiatus band in Waterfall view to rename it or change its highlight color for that week.

Notes (in the preview)

The Notes column auto-fills key milestones inline — e.g. “Start Principal Photography 11/02/26”, “Principal Photography Wraps”, union holidays, and any simultaneous-post flags.

The two views own their notes separately. Waterfall notes are the master set: they appear in the Waterfall, the Excel export, and the Month view. Month view notes — added by hovering a day and clicking a + — belong to the Month view and its PDF only, and deliberately do not appear in the Waterfall or the Excel export. Every day block shows at least four lines, and each free line has its own +, so a note goes exactly where you click rather than always dropping to the bottom. A note stays on the line you added it to. If a day needs more room than the phases leave, the wider + on the bottom edge of the block adds another line — to the whole week, since the seven day blocks share one calendar row — up to a maximum of six. A block always grows to fit its actual content, so nothing is ever hidden even past that maximum.

Every month prints on exactly one page, filling the sheet. Each page is pinned to the paper and the week rows divide it between them: an empty row still gets its four lines, so a sparse month prints as uniform full-size rows rather than a small grid floating at the top. A row carrying extra comments takes proportionally more of the page and the others give way, so everything stays visible and the calendar always spans top to bottom. If the grid lines print faint or missing, switch on Background graphics in the print dialog’s More settings. Waterfall notes can be edited from either view — editing one in Month view updates the Waterfall and the Excel export too, since it’s the same note. Because a Waterfall note covers a whole week, that editor shows the week’s whole text, so the other notes in it aren’t lost. In Waterfall view, click a note cell to open an editor. Type to replace it, or clear it completely to remove a pre-filled milestone — a cleared note stays gone and won’t regenerate. An empty note loses its highlight. Because a waterfall row covers a whole week, its editor has a Day dropdown. Picking a day stamps that date into the note text — “Table Read” becomes “Table Read 11/04/26” — matching how the automatic notes are written. The date is ordinary text you can reword or delete, and it’s what places the note in Month view; switching days rewrites it rather than adding another. Leave the dropdown on “Mon (default)” and the note simply sits on that week’s Monday. Notes added in Month view are pinned to the day you clicked, and each keeps its own highlight colour. The editor also shows the ten standard Microsoft Excel colors — pick one to change that individual cell’s highlight color (text flips black or white automatically for readability). Press Tab or Ctrl/Cmd‑Enter to save, Esc to cancel, or just click away. The Reset Notes & Hiatus button above the header returns every note, holiday, and hiatus band to its auto-generated text and default color — including bringing back any milestones you deleted. (That’s different from Reset All in the toolbar, which clears the whole calendar and asks first.)

Waterfall view vs. Month view

Each view has its own header with its own Auto/Manual switch, and the two are independent — taking the month header manual leaves the waterfall header alone, and vice versa. In Auto the lines fill themselves from Show Info; switch to Manual and they become an editable snapshot you can word however you like.

Toggle at the top of the preview. Waterfall view mirrors the Excel export — a week-by-week grid with editable notes — and is the default. Month view is a day-by-day calendar where phases run as bars across the days they cover; Production shows as individual episodes.

Exporting

In Month view the button becomes “Export Calendar to PDF.” It opens your browser’s print dialog — choose Save as PDF as the destination. Every month from the first phase to the last prints, one per landscape page, in full colour with the header, month and date, exactly as it looks on screen (minus the paging arrows). If the colours come out grey, switch on “Background graphics” in the dialog’s More settings.

Generates a styled .xlsx matching the Waterfall view: one column block per year, colored phase cells, the milestone Notes column, and a bold printed header with the title, date, and production summary. Columns are sized tight to their content, and the whole grid is scaled to print as large as possible on a single sheet. Orientation is chosen automatically — calendars of two years or fewer print portrait, three or more years print landscape — unless the other orientation would fit noticeably larger. The file name uses your show title (e.g. “Show Title - Season 1 Planning Calendar.xlsx”).

Files, saving & reopening

All the main actions live in the toolbar at the top: the file menu (showing the calendar you’re currently in), New, Save, Save As, Export to Excel, and Reset.

Save writes a self-contained copy of this tool with all your entries baked in (title, dates, phases, custom phases, hiatuses, edited notes, header mode, and current view). In Chrome or Edge the first save asks where to put the file, and every save after that writes straight back to that same file with no dialog. Open the saved file later and everything comes back exactly as you left it.

New clears everything and starts a blank calendar. Save As is different — it copies the calendar you’re working on into a new file and switches you to it, which is handy for forking a schedule to try a variation without touching the original.

The file menu lists calendars you’ve saved; click one to switch to it, use Open… to pull in a calendar file that isn’t listed, or the × to drop a file from the list (that removes the shortcut only, never the file itself). Reopening a file may show a quick one-click permission prompt (“Let this site edit the file?”) — that’s the browser protecting your disk. The file menu and in-place saving need Chrome or Edge; in other browsers each save downloads a new copy instead.

Autosave & recovering work

The indicator beside Export shows Unsaved changes or the time of your last save. Once a calendar is linked to a file, it autosaves every 10 minutes in the background — no dialog, nothing to click.

Work that hasn’t been saved to a file yet can’t autosave (browsers won’t create a file without you choosing where it goes), so the app keeps a rolling backup in your browser’s local storage instead. If a session ends with unsaved changes, the next time you open the app it offers to recover that work. Closing the tab with unsaved changes also triggers your browser’s “leave site?” warning.

The backup lives only in this browser on this computer — it’s never uploaded anywhere, and clearing your browsing data removes it. Treat it as a safety net; the saved file on your disk is still the real copy.

Reset

Reset (bottom of the panel) clears all your entries and returns every setting to its default state. Two narrower resets live above the calendar: Reset Notes & Hiatus restores just the notes, holidays, and hiatus bands, and Header: Switch to Auto returns the header to live auto-fill.

Tip: any date you type snaps to the Monday of its week, and all durations are honored in full even when holidays or hiatuses interrupt them.
